# STProof 协议白皮书

**版本 1.3 | 2026年3月27日**


**摘要**
STProof (Spatial-Temporal Proof) 是一套面向“
**事件**”的通用链上确权与价值捕获协议。它旨在解决物理世界与数字世界中，那些**无法通过标准化API验证**的独特历史事件的真实性记录难题。


**核心价值**：在AI伪造技术泛滥的时代，为“**事实**”提供一个去中心化、不可篡改、可交叉验证的永久锚点。


**关键创新**：

*   **确权对象**：从“资产”转向“**事件**”。实物仅为事件载体，NFT只锁定事件本身，彻底规避RWA（现实世界资产）监管风险。

*   **验证范式**：采用“**多源证据交叉验证**”模型，要求至少两个独立来源的证据（如GPS、官方记录、影像），并经由社区或付费验证者网络确认。

*   **价值机制**：首创“**引用税**”模型。事件的价值不再依赖二级市场炒作，而是与其被链上引用的次数挂钩，实现价值与影响力的正循环。

*   **可扩展性**：通过“**事件模板**”系统，允许社区为不同领域（如集邮、艺术品、新闻、法律）定义自定义的验证规则，使协议具备无限扩展性。


**核心理念**：

*   **不确权物，只确权事**。实物归实物世界，事件归区块链世界。

*   **事实的记录不应有资金门槛**。任何人仅需支付网络Gas费即可铸造事件凭证。

*   **凡发生，皆可存；凡存证，皆可信**。

## 1. 引言：物理世界事实的记录难题
在信息爆炸与AI深度伪造技术并行的今天，
**事实的“真实性”** 正面临前所未有的挑战。传统中心化记录方式（公证、档案）成本高、易篡改；而现有区块链方案大多仅锚定数字文件本身，缺乏与物理世界事件的刚性连接。

一枚首日封是否真的在发行首日被寄出？一幅画作的创作过程是否真实？这些
**无法通过API回答**的非标事件，恰恰构成了人类历史与文化的重要部分。STProof协议正是为了填补这一空白。


## 2. STProof：事件确权协议

### 2.1 定义
STProof是一套将
**客观发生的历史事件**转化为链上可信凭证的协议。其确权对象统一为“**事件**”，而非任何实体或数字资产。


### 2.2 核心五要素
每个被确权的事件都必须包含以下五个维度的信息：


**表1：事件确权核心五要素**

| 要素 | 说明 | 有实物案例（首日封） | 无实物案例（线上发布） |
| :--- | :--- | :--- | :--- |
| 
**时间** | 事件发生的精确时间点 | 2026-03-24 11:24 | 2026-03-24 15:30 UTC |
| 
**空间** | 事件发生的真实地理位置 | 香港天水围邮局 (114.004° E, 22.451° N) | 特定URL或链上地址 |
| 
**主体** | 事件的参与者（人或物） | 寄件人、收件人 | 作者、发布平台 |
| 
**事实** | 事件发生的具体行为内容 | “X号首日封以挂号信寄出” | “文章《Y》首次发布” |
| 
**证据** | 证明事实发生的多源材料 | GPS轨迹、邮政收据、现场影像 | 链上时间戳、数字签名、第三方存档快照 |


### 2.3 可扩展基础：事件模板
协议通过“
**事件模板**”实现无限扩展。社区可以为特定类型的事件定义：

1.  **证据要求**：需要哪些维度的证据。

2.  **验证者画像**：谁有资格验证此类事件。

3.  **引用场景**：预期在何种场景下被引用。

4.  **可选模块**：是否启用实物存留机制。

模板需经社区治理通过，其创建者可获得该模板后续产生的部分费用分成。


### 2.4 证据标准与存储

*   **多源独立性**：证据需来自至少两个独立来源。

*   **可信时间戳**：证据包需经国家认可的时间戳机构（如联合信任）固化。

*   **去中心化存储**：证据包上传至IPFS/Arweave，获得内容标识符（CID）。

*   **分层披露**：证据分为“公开包”（免费浏览）和“完整包”（付费访问），兼顾透明性与数据价值。


### 2.5 铸造与验证机制

*   **平权铸造**：任何人仅需支付网络Gas费即可铸造Event NFT。

*   **双轨验证**：

    *   **社区志愿验证**：免费，由社区验证者进行，速度较慢。

    *   **激励验证**：发起人支付费用，由专业验证者快速验证。

## 4. 经济模型：平权优先与价值捕获
本模型旨在通过
**零铸造费**降低参与门槛，并通过**引用税**让事件价值回归其社会影响力。


### 4.1 核心角色与收益

**表3：协议参与者角色与收益**

| 角色 | 贡献 | 收益来源 |
| :--- | :--- | :--- |
| 
**事件发起人** | 采集证据、铸造NFT | 1. 转售版税（5%）<br>2. 链上引用费（70%）<br>3. 付费访问费（70%） |
| 
**验证者** | 验证事件真实性 | 1. 声誉积分（志愿）<br>2. 验证费ETH（激励） |
| 
**模板创建者** | 设计新事件类型的验证规则 | 模板使用费分成 |
| 
**协议金库** | 维护与发展协议 | 引用费与激励验证费的30%分成 |


### 4.2 三层价值捕获机制

**表4：三层并行价值捕获机制**

| 收入来源 | 触发条件 | 费率/金额 | 分配 | 价值内涵 |
| :--- | :--- | :--- | :--- | :--- |
| 
**转售版税** | NFT在二级市场转售 | 成交价的5% | 发起人100% | 捕获事件的**稀缺性**与收藏价值 |
| 
**链上引用费** | 被其他地址/合约引用 | 建议0.0001 ETH/次 | 发起人70%<br>金库30% | 捕获事件的**影响力**与使用价值 |
| 
**付费访问费** | 用户下载完整证据包 | 0.001 ETH/次或订阅 | 发起人70%<br>金库30% | 捕获事件的**知识传播**价值 |


**链上引用是核心创新**。每次引用都是对事件真实性的公开背书，引用次数成为衡量其社会公信力的透明指标。引用场景包括学术研究、新闻报道、司法证据、社交媒体声明等。


## 5. 智能合约实现
协议基于以太坊，采用ERC-721标准，并扩展以下功能：

*   **EIP-2981版税**：自动分配转售版税。

*   **动态元数据**：支持通过 `setTokenURI` 更新事件状态。

*   **链上引用**：通过 `cite()` 函数记录引用并分配费用。

核心接口摘要：
 
 
solidity
interface ISTProof {
function mint(address to, uint256 templateId, string memory uri) external;
function mintWithIncentive(address to, uint256 templateId, string memory uri, address[3] memory validators) external payable;
function cite(uint256 tokenId) external payable;
function getCitationCount(uint256 tokenId) external view returns (uint256);
}

## 6. 首个案例：首日封实寄事件确权
**事件**：2026年3月24日，一枚“虚白斋藏品”首日封于香港天水围邮局以挂号信寄出。
**证据包**：
*   **时间**：邮局时钟、收据、GPS日志交叉验证。
*   **空间**：34个GPS路点形成的精确轨迹。
*   **行为**：购买、填写、盖戳、投递全程影像记录。
*   **官方记录**：挂号信收据（追踪号RE176993145HK）及官网查询截图。
*   **固化存储**：所有证据经联合信任时间戳固化后，分层存储于IPFS。

**意义**：成功验证了协议对“有实物事件”全生命周期进行可信记录的可行性，并为其他领域提供了可复制的模板。

## 8. 协议边界与合规声明
### 8.1 协议不适用于
*   纯主观感受或个人观点。
*   无法被任何第三方验证的私密事件。
*   协议不保证实物本身的物理真实性（需依赖传统鉴定）。
*   协议不提供实物托管、保险或法律意见服务。

### 8.2 合规定位
STProof明确自身为**非金融基础设施**：
*   **非RWA**：NFT不锚定实物所有权或收益权。
*   **非证券**：不发行代币，无募资与分红承诺。
*   **司法友好**：采用多源证据+可信时间戳+去中心化存储，符合中国及多地法院对区块链电子证据的认定标准。

## 核心术语表
*   **事件确权**：将客观发生的历史事件本身，而非关联的资产，作为确权对象。
*   **引用税**：其他智能合约或用户为引用某个事件NFT的链上记录而支付的小额费用，是事件价值的主要来源。
*   **事件模板**：由社区定义的、针对某一类事件（如“首日封实寄”）的标准化验证规则集。
*   **多源交叉验证**：要求来自至少两个独立来源的证据，且证据间能通过唯一标识符相互印证，形成闭合证据链。
*   **CID**：内容标识符，是存储在IPFS等去中心化网络上的数据的唯一哈希地址，保证内容不可篡改。
